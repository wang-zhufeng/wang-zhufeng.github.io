项目背景与研究问题

地下煤矿甲烷爆炸是全球最严峻的工业安全威胁之一。现有监控系统以单一阈值报警为主，面对多变量气体-风速-温度的复合前兆信号时存在严重盲区——运营人员经常面对「报警洪水」却无法快速定位根因。

核心研究目标

本项目聚焦于 Triple-Correlation Analysis（TCA）框架的可解释可视化层设计，而非优化算法本身。目标是将数值精确但认知不透明的相关性矩阵，转化为操作人员无需统计学背景即可实时理解的动态网络视图。

核心问题：如何在时间压力下，将复杂多变量气体相关性分析结果设计成操作人员可信赖、可操作的可视化界面？

数据集
  • 来源：中兴矿真实传感器数据（Wu et al., 2021）
  • 规模：12个传感器通道 × 5,749 时间步
  • 类型：气体（FS1–FS4）、温度（T1–T8）、风向（WD1–WD2）
  • 周期：2022年1–2月，约320万条独立读数
  • 识别稳定三元组：197条（FSV过滤后，存活率69.4%）

三版迭代演进

系统历经三个版本的完整重构，每一版都基于前一版的可用性缺陷做出根本性设计决策调整。

V1 · Week 1–6
继承矩阵系统 — 修复与增强
接手现有三分法代码库，发现四项严重缺陷：KMO 值硬编码为常数 0.500、矩阵单元格仅显示二值颜色无数值、相关阈值固定不可调、无数据清洗流水线。逐一修复：动态 KMO 计算、嵌入 r 系数数值、增加 RELAXED/STRICT 模式切换与 r-min 滑块、添加 IQR 异常值检测 + z-score 归一化预处理流水线。矩阵范式的根本局限仍在——操作人员仍需扫描行列才能推断拓扑关系。
V2 · Week 7–11
网络图系统 — 从零重构
核心洞察：矩阵以表格坐标编码关系，强迫用户在头脑中重建拓扑；而动态多变量网络（DMVN）直接外化拓扑——传感器变为节点，相关性变为边，图结构本身传达哪些传感器是核心、哪些是外围。接受 CSV 流替代 Excel 文件，引入参数控制面板（|r| / p / 分析深度），绿色表示正相关、红色负相关。Week 11 升级：边渲染同时通过线宽和颜色饱和度编码五级相关强度，并添加鼠标悬停高亮和方向动画。
V3 · May 2026
UGMAS 全面板仪表盘 — 最终系统
将抽象浮动节点布局替换为空间锚定矿井地图，传感器节点定位于真实物理坐标。新增气泡墙可视化面板（气泡半径编码三领域相关密度）、LSTM 一小时预测轨迹（T1–T8全部传感器）、为工程师和操作人员双用户角色设计的数据输入管道视图（JSON面板 + 平铺传感器值表），以及 NORMAL/ALERT 实时状态指示器。
三层系统架构
  • 数据层：CSV 采集 → 模式验证 → IQR异常值筛除 → z-score/min-max归一化 → NumPy矩阵序列化为JSON
  • 应用层（Python/Flask）：TCA 三元组相关系数计算（几何均值Pearson）→ FSV 稳定性验证（30次Bootstrap ±5%漂移门控）→ RESTful API
  • 视图层（Vue.js + D3.js）：DMVN 动态网络渲染 → 参数控制面板（|r| / p / 深度）→ 矿井地图 → 气泡墙 → LSTM预测图
TCA 算法核心

对12传感器池中每个有序三元组 (Xᵢ, Yⱼ, Zₖ) 计算复合相关系数：三对 Pearson 系数绝对值的几何均值归一化至 [-1,+1]，双尾 t 检验 α=.01 评估显著性。

五级强度解释尺度：Great (|r|≥.90) · Very Good (.75–.89) · Good (.50–.74) · Fair (.30–.49) · Poor (.00–.29)

最强发现：FS1–T7–T8 三元组 r=0.93，FS2–T6–T7 r=0.91，气温配对主导高强度层，与地下甲烷通过浮力驱动分层积累的物理机制一致。

用户评估结果

5名研究生工程专业学生，DMVN可视化 vs 静态相关矩阵根因溯源对比测试。使用 DMVN 视图的参与者一致更快完成任务，自报信心评分在5人中均高于矩阵条件，印证了动态网络视图在安全关键场景下降低认知负荷、加速决策的文献结论。
